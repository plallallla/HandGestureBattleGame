# AR Hand Gesture Battle Game

这是一个基于计算机视觉的增强现实（AR）猜拳互动游戏。玩家通过摄像头比出不同的手势，与屏幕上出现的“敌人”进行实时对战。项目集成了 **OpenCV** 进行视频流处理，**MediaPipe** 进行高性能手势识别，以及 **Pygame** 处理游戏渲染与逻辑控制。

---

## 核心架构设计

项目采用模块化设计，主要分为三个核心部分：

### 1. 视觉感知层 (`GestureDetector` 类)
负责将原始摄像头画面转换为可理解的游戏指令。
*   **MediaPipe 集成**：使用 `HandLandmarker` 进行 21 点关键点检测。
*   **手势识别**：基于指尖与关节关键点的 Y 轴坐标对比，定义了 `rock` (石头)、`paper` (布)、`scissors` (剪刀) 的分类逻辑。
*   **稳定性优化**：
    *   **滑动窗口过滤**：引入长度为 3 的历史记录队列，只有连续 3 帧识别结果一致才判定为有效手势，有效消除抖动误判。
    *   **性能优化**：通过 `frame_count % 3 == 0` 机制，每 3 帧处理一次检测，平衡 CPU 占用与实时性。
*   **镜像处理**：在逻辑层对摄像头画面进行水平翻转（镜像处理），符合人类直觉交互。

### 2. 游戏引擎层 (`HandGestureBattleGame` 类)
采用有限状态机（FSM）管理游戏生命周期：
*   **`START`**: 初始界面，等待玩家比出“剪刀”手势触发游戏开始。
*   **`READY`**: 倒计时准备阶段（2 秒），给玩家缓冲时间。
*   **`RUNNING`**: 游戏循环阶段，负责图标生成、逻辑判定、HP 和分数计算。
*   **`GAME_OVER`**: 结算阶段，保存游戏截图并更新排行榜。

### 3. 互动逻辑层
*   **智能刷怪 (`spawn_icon`)**: 采用了避障算法，优先在屏幕空闲区域生成敌人，并避免连续两次在同一象限刷新。
*   **胜负判定**: 
    *   采用象限碰撞检测（Hand Quadrant == Icon Quadrant）。
    *   严格校验左右手（颜色编码：红色=左手，绿色=右手）。
    *   自动反转手势逻辑（如：敌人出石头 -> 判定玩家是否出布）。

## 技术细节实现

### 游戏参数配置
可通过修改常量直接调整难度：
*   `HP_MAX`: 玩家初始生命值。
*   `ICON_LIFETIME`: 敌人存在时间（超时则扣血）。
*   `SPAWN_INTERVAL`: 刷怪频率。

### 碰撞判定逻辑
游戏将屏幕分为四个象限（Top-Left, Top-Right, Bottom-Left, Bottom-Right）。碰撞判定并不依赖像素精确重叠，而是基于手势中心坐标所在的象限与图标象限的一致性，极大提升了互动流畅度。

### 排行榜与截图系统
*   **持久化存储**：使用 `leaderboard.json` 存储分数数据。
*   **视觉留念**：游戏结束时触发 `cv2.imwrite` 保存当前帧快照，并在 `GAME_OVER` 界面通过 `pygame.image.frombuffer` 展示。

## 环境要求

在运行代码前，请确保安装了以下依赖：

```bash
pip install opencv-python mediapipe pygame
```

*注意：还需要确保目录中存在 `hand_landmarker.task` 模型文件，MediaPipe 需要该文件进行推理。*

---

## 如何游玩

1.  **启动游戏**: 运行代码后，摄像头窗口将开启。
2.  **开始游戏**: 在摄像头前比出“剪刀”手势。
3.  **对战**:
    *   **左手/右手**: 观察屏幕上的图标颜色（红色图标需用左手，绿色图标需用右手）。
    *   **手势**: 出对应的克制手势（如敌人出石头，你需要出布）。
4.  **失败判定**: 如果操作超时、手势错误或用错手，将会扣减 HP。
5.  **结束**: HP 归零后，按 `空格键` 重开，`ESC` 退出。

## 扩展建议
*   **难度曲线**: 可以随着 `score` 的增加，减少 `ICON_LIFETIME` 以增加游戏难度。
*   **音效**: 集成 `pygame.mixer` 增加打击感和胜利/失败音效。
*   **图形资源**: 将简单的 `Rect` 绘制替换为加载 `.png` 图标文件，提升 UI 表现力。
